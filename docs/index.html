Axiomatizations of Priority-Based Ranking Rules

A hiring committee evaluates candidates by publications across journal tiers.

Top-5   \(>_p\)   A*   \(>_p\)   A   \(>_p\)   B   \(>_p\)   C


How many B-publications equal one Top-5?

A university compares fundraising campaigns

by projected revenue across priority areas.

Medicine   \(>_p\)   Engineering   \(>_p\)   Humanities

Campaign X: $2M from medicine alone.   Campaign Y: $1M from each — $3M total.


Should a higher total from lower-priority areas beat a smaller sum from the top?

A hospital ranks infection control protocols

by performance across patient risk categories.

ICU   \(>_p\)   Surgical   \(>_p\)   General ward


Is a protocol with some protection in every category better

than one that’s excellent in ICU but leaves general wards unaddressed?

The standard approach


Assign weights to each dimension.

Compute a weighted sum.

Rank by the number.



Requires choosing weights—

an inherently arbitrary step.

Assumes commensurability

across dimensions.

The axiomatic approach


State the principles you believe in.

Derive the unique rule they imply.

The rule justifies itself.



No weights to choose.

No commensurability assumed.

Normative transparency.

Barberà and Jackson (JET, 1988)


Axiomatically characterized three ranking rules

for real vectors without weights.


Rule How it works Philosophy
Maximin Compare \(\min_i x_i\) Worst coordinate decides
Leximin Sort, then compare lexicographically Improve the worst off first
Protective Which thresholds are met, by how many Breadth of threshold coverage

Three rules, each justified by axioms — no weights needed.

But their framework assumes

all dimensions are symmetric.



It cannot distinguish a Top-5 from a B.

It cannot distinguish medicine funding from humanities funding.

It cannot distinguish ICU protection from general ward protection.

This paper


extends the axiomatic approach to settings where

dimensions carry an exogenous priority order.



Six rules.   Three families.   Three philosophies.


Each characterized by a small set of axioms

with independent normative content.

Three families

Quality-first

Only top-tier performance matters

PMM · PLS

No lower-tier output offsets a higher-tier deficit

Quantity-first

Aggregate output matters most

TMM · TLS

Trade-offs across tiers are permitted

Level-first

Which thresholds are met matters

P-PROT · Q-PROT

Advantages at one threshold cannot offset others

The six rules


Rule Philosophy What it compares Decision
■ PMM B&J Quality-first Top coordinate \(x_1\) Highest-priority score wins
■ PLS B&J Quality-first \((x_1, x_2, \ldots, x_n)\) First difference decides
■ TMM new Quantity-first Total \(S_n = \sum x_i\) Highest total wins
■ TLS new Quantity-first \((S_n, S_{n-1}, \ldots, S_1)\) Total first, then prefix sums
■ P-PROT B&J Level-first Coverage sets \(H_a\) Lowest differing threshold
■ Q-PROT new Level-first Coverage sets \(H_a\) Highest differing threshold

\(H_a(x) = \{i : x_i \geq a\}\) — the set of coordinates that “meet threshold \(a\)”

All six rules on one example

\(n = 3\)   \(x = (3, 1, 0)\)   \(y = (2, 0, 3)\)

Prefix sums:   \(S(x) = (3, 4, 4)\)   \(S(y) = (2, 2, 5)\)

Quality-first

PMM: \(x_1 = 3 > 2\) → \(x\)

PLS: first diff at 1 → \(x\)

\(x\) wins — top-tier strength

Quantity-first

TMM: \(S_3 = 4 < 5\) → \(y\)

TLS: \(k^* = 3\), same → \(y\)

\(y\) wins — higher total

Level-first

P-PROT: \(a^*\!=\!1\), \(r^*\!=\!2 \in H_1(x)\)

Q-PROT: \(a^*\!=\!3\), \(r^*\!=\!1 \in H_3(x)\)

\(x\) wins — coverage breadth / peak excellence

From rules to axioms


Each rule is uniquely determined by a small set of principles

that a decision-maker can directly evaluate.



Not “this is the rule.”

But rather: “if you believe these principles,

then this is the only rule consistent with them.”

The standard axioms

Axiom Meaning
C — Completeness All pairs comparable:   \(x \succeq y\) or \(y \succeq x\)
T — Transitivity Consistent rankings:   \(x \succeq y,\; y \succeq z \Rightarrow x \succeq z\)
PD — Pareto Dominance \(x_i \geq y_i\;\forall i,\; x_j > y_j\) some \(j \;\Rightarrow\; x \succ y\)

Monotonicity and indifference

TCM — Top-Coordinate Monotonicity

\(x_1 > y_1 \;\Rightarrow\; x \succ y\)

Higher top-coordinate is strictly better

TCI — Top-Coordinate Indifference

\(x_1 = y_1 \;\Rightarrow\; x \sim y\)

Same top-coordinate means indifference

TM — Total Monotonicity

\(S_n(x) > S_n(y) \;\Rightarrow\; x \succ y\)

Higher total sum is strictly better

TI — Total Indifference

\(S_n(x) = S_n(y) \;\Rightarrow\; x \sim y\)

Same total sum means indifference

The compensation axioms


NCA — Non-Compensability Across Levels

If \(x_k > y_k\) at the first differing coordinate, no amount of advantage at coordinates \(k+1, \ldots, n\) can make \(y\) strictly beat \(x\).

“No lower-tier excellence compensates for a higher-tier deficit.”

GUT — Generalized Upward Transfer

Redistributing mass from coordinate \(k+1\) to any combination of \(1, \ldots, k\) (preserving the total) is strictly better.

“Concentrating output at higher-priority dimensions is always better.”

The compensation axioms (continued)


NUTC — No Upward Threshold Compensation

A coverage advantage at a low threshold cannot be offset by differences at higher thresholds.

“Safety first”: meeting basic thresholds broadly matters most.

NDTC — No Downward Threshold Compensation

A coverage advantage at a high threshold cannot be offset by differences at lower thresholds.

“Excellence first”: meeting the highest thresholds matters most.

The characterization theorems


Rule \(=\) Axioms Key axiom
■ PMM TCM + TCI Only top coordinate matters
■ PLS C + T + PD + NCA No compensation across tiers
■ TMM TM + TI Only total matters
■ TLS C + TM + GUT Upward transfer is better
■ P-PROT C + T + PD + NUTC Safety first
■ Q-PROT C + T + PD + NDTC Excellence first

Each characterization is an if and only if.   All axioms are independent.

Impossibility

The foundation axioms are pairwise incompatible.

No single preference can straddle two families.

TCM TM NUTC NDTC
TCM ✗ ✗ ✗
TM ✗ ✗ ✗
NUTC ✗ ✗
NDTC ✗ ✗

Each ✗ is a theorem: no preference on \(\mathbb{R}^n\) satisfies both axioms simultaneously.

The deeper question: commensurability

Dimensions commensurable

Same currency, different quality levels

Publications across journal tiers

Prefix sums are meaningful:

“total output at tier \(j\) or above”

FOSD-respect is desirable


→ Quality-first or Quantity-first

Dimensions incommensurable

Qualitatively distinct dimensions

Failure modes by criticality

Prefix sums aggregate unlike quantities:

no normative meaning

FOSD violation is a feature


→ Level-first

A hidden fault line

In B&J’s symmetric setting, Maximin and Leximin sort coordinates before comparing — implicitly treating values across dimensions as commensurable.

The Protective criterion uses coverage sets — never aggregating across dimensions.

Under symmetry, dimensions have no identity — so this difference cannot be tested.

A hidden fault line (continued)

Drop symmetry, and the fault line becomes consequential:

  Quality-first and Quantity-first rely on prefix sums — dimensions must be commensurable.

  Level-first uses coverage sets — no aggregation across dimensions.

The commensurability question, latent in B&J, becomes the organizing principle.

Example: NCA and TM are incompatible

The full proof in Lean 4

/-- NCA and TM are incompatible (proved for n = 2). -/
theorem NCA_TM_impossible (R : PrefRel 2) :
    Ax_NCA R → Ax_TM R → False := by
  intro hNCA hTM
  let x : Vec 2 := ![0, 10]
  let y : Vec 2 := ![1, 0]
  -- TM gives x ≻ y since totalSum x = 10 > 1 = totalSum y
  have hsum : totalSum x > totalSum y := by
    simp only [totalSum, x, y]
    rw [Fin.sum_univ_two, Fin.sum_univ_two]
    norm_num
  have hTMxy : strictPart R x y := hTM x y hsum
  -- NCA forbids x ≻ y
  have hvac : ∀ i : Fin 2, i < (0 : Fin 2) → y i = x i := by
    intro i hi; exact absurd hi (not_lt.mpr (Fin.zero_le i))
  have hgt : y (0 : Fin 2) > x (0 : Fin 2) := by
    simp [x, y, Matrix.cons_val_zero]
  exact (hNCA y x 0 hvac hgt x (fun _ _ => rfl)) hTMxy

Constructs \(x = (0, 10)\), \(y = (1, 0)\):   TM forces \(x \succ y\);   NCA forbids it.
